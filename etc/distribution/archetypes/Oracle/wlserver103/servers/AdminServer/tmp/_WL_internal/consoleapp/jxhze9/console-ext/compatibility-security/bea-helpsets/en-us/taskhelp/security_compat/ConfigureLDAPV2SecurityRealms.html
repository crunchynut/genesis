Configure LDAP V2 security realms


In Compatibility security, the LDAP V2 realm is configured as a custom security realm.

The LDAP tree and schema is different for every LDAP server. WebLogic Server includes templates for the supported LDAP servers. These templates specify default configuration information used to represent users and groups in each of the supported LDAP servers.

To use a LDAP V2 realm:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the left pane, expand Compatibility Security > Realms and click New.

    The Create a New Basic Realm page appears.
  3. In the Type drop-down list, select LDAP Realm V2 and click OK.
  4. Define attributes for the LDAP directory server and specify how users and groups are located in the LDAP V2 security realm.
  5. When you have finished defining all the attributes, reboot WebLogic Server.
  6. Configure the Caching realm. For more information, see Configure Caching realms . When configuring the Caching realm, select the LDAP Realm V2 option from the pull-down menu for the Basic Realm attribute on the Caching Realms: Configuration: General page. The Basic Realm attribute defines the association between the Caching realm and the alternate security realm (in this case, the LDAP V2 security realm).
  7. In the left pane, select your domain (for example, mydomain).
  8. Select Compatibility Security > File Realm.
  9. In the Caching Realm field, choose the name of the Caching realm to be used with the LDAP V2 security realm. A list of configured Caching realms appears on the drop-down menu.
  10. Reboot WebLogic Server.
