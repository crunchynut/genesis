Caching Realm: Users

Configuration Options     Related Tasks     Related Topics

Use this page to configure and enable the Users cache for the Caching realm. To use the Caching realm, you need to run Compatibility security. The use of the Caching realm is deprecated in this release of WebLogic Server.

Configuration Options

Name Description
Enable User Cache

Specifies whether the User cache should be enabled. By default, the User cache is enabled.
User Cache Size

The maximum number of user lookups (between 17 and 65537) that should be cached.

This attribute should be a prime number for best performance.
User Cache TTLPositive

The number of seconds (between 1 and a positive 32-bit integer) to retain the results of a successful user lookup.
User Cache TTLNegative

The number of seconds (between 1 and a positive 32-bit integer) to retain the results of an unsuccessful user lookup.
